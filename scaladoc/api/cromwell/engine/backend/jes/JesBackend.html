    Calls in Running *with* a job key should be left in Running.

    • Any execution in Failed should fail the restart.
    • Any execution in Aborted should fail the restart.
    • Scatters in Starting should fail the restart.
    • Collectors in Running should be set back to NotStarted.
    • Calls in Starting should be rolled back to NotStarted.
    • Calls in Running with no job key should be rolled back to NotStarted.

    Calls in Running *with* a job key should be left in Running. The WorkflowActor is responsible for resuming the CallActors for these calls.
